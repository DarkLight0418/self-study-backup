# 네트워크 손필기 전사용 노트 생성 프롬프트 템플릿

## 프롬프트 본문

```md
아래 자료들을 바탕으로 [챕터명/범위] 손필기 전사용 노트를 만들어주세요.

참고 자료:

- 교안 PDF: [파일명]
- 손필기 사진(있다면): [파일명들]
- 이전 버전 참고본(있다면): [ver1/ver2/ver3/ver4/ver5 파일명들]

이번 작업의 핵심 목표는 다음과 같습니다.

1. **개념별 정리 구조**를 기본 뼈대로 유지해주세요.
   - 즉, 페이지 순서만 그대로 나열하지 말고
   - 큰 개념 단위(예: UDP, TCP, RDT, Flow Control, Congestion Control, DNS, HTTP 등)로 묶어서 정리해주세요.

2. 다만 각 개념 제목 옆에는 반드시 **관련 페이지 범위**를 적어주세요.
   - 예: `## 3. TCP Flow Control (p89 ~ p95)`
   - 즉, "개념별 구조 + 페이지 감각"이 동시에 보이게 해주세요.

3. **직접 손필기한 표현**을 적극 반영해주세요.
   - 손필기 메모에 있는 표현이 이해와 암기에 유리하면 살려주세요.
   - 예: "집 주소만으로는 부족하고 누구 방인지까지 알아야 함"
   - 예: "한번 열고 쫙 보냄"
   - 예: "안 바뀐 파일은 다시 안 보내서 시간과 트래픽 절약"

4. **수업식 강조 포인트**를 넣어주세요.
   - 교수님이 중요하게 다루고 지나간 개념
   - 시험에서 비교형/서술형/단답형으로 자주 나올 만한 부분
   - 헷갈리기 쉬운 비교 포인트를 따로 표시해주세요.

5. **임의로 내용을 빼지 말아주세요.**
   - 손필기 메모에 중요하게 적혀 있거나
   - 수업에서 중요하게 다뤘던 개념이면
   - 챕터가 길어도 빠뜨리지 말고 포함해주세요.
   - 특히 [중요 단원/예: TCP 심화, RTT, flow control, congestion control, handshake]는 절대 빼지 말아주세요.

6. 출력 결과는 **손으로 전사하기 쉬운 문장 길이**로 작성해주세요.
   - 한 줄이 너무 길지 않게
   - 짧은 문장, 짧은 bullet, 화살표식 설명을 많이 사용해주세요.
   - 실제 공책/노트에 옮겨 적기 쉽게 해주세요.

7. 처리가 필요한 그림/도식 부분은 **md에서 볼 수 있는 ASCII 와이어프레임 또는 표 형태**로 바꿔주세요.
   - 즉, "그림 그리세요"라고만 하지 말고
   - 내가 보고 그대로 손으로 베껴 그릴 수 있게
   - 박스/화살표/간단한 흐름도를 텍스트로 만들어주세요.

8. 결과물의 구성은 아래 형식을 따라주세요.

---

# [챕터명] 손필기 전사용 노트

## 목적

- 시험 직전 복습 가능
- 손으로 전사 가능
- 교안 + 손필기 + 수업 강조 포인트 통합

## 이 노트의 사용 방식

- 먼저 외울 핵심 문장
- 개념별 정리
- 손필기식 표현
- 시험 포인트
- 비교 정리
- 와이어프레임/도식
- 마지막 체크리스트

# 0. 먼저 외울 핵심 문장

- 챕터 전체를 관통하는 핵심 문장 10~20개

# 1. [개념1 이름] (p** ~ p**)

## 1-1. 정의 / 큰 그림

## 1-2. 손필기식 표현

## 1-3. 시험 포인트

## 1-4. ASCII 와이어프레임 또는 표

# 2. [개념2 이름] (p** ~ p**)

...

# 마지막. 헷갈리는 비교 정리

- 예: TCP vs UDP
- 예: Flow Control vs Congestion Control
- 예: GBN vs SR
- 예: Non-persistent vs Persistent HTTP
- 예: SMTP vs IMAP
- 예: Iterative vs Recursive DNS Query

---

작성 스타일 규칙:

- 지나치게 교과서식으로 딱딱하게 쓰지 말고
  **직접 손으로 필기하기 좋은 설명체**로 써주세요.
- 단, 너무 구어체로 무너지지 말고
  **시험 대비용 개념 정확성**은 유지해주세요.
- 필요하면 비교표를 넣어주세요.
- 필요하면 "왜 중요한가", "무엇과 다른가", "왜 이런 설계가 필요한가"를 한 줄씩 붙여주세요.
- 수식이 나오면 수식 + 의미 + 왜 필요한지까지 같이 적어주세요.
- 프로토콜 흐름이 나오면 송신자/수신자/라우터/서버 역할이 헷갈리지 않게 적어주세요.
- 코드가 필요 없는 챕터면 코드 없이 정리해주세요.
- 코드 예제가 있는 챕터라도, 손필기 전사용이면 **코드 전문보다 개념과 흐름 중심**으로 요약해주세요.

출력 형식:

1. 먼저 완성된 `.md` 본문을 작성
2. 이어서 `.txt`로도 바로 저장 가능한 형태를 제공
3. 가능하면 파일도 생성해주세요.

중요:

- 이전 버전보다 임의로 빠진 개념이 없도록 해주세요.
- 손필기 메모에 적힌 표현이 이해에 도움 되면 최대한 반영해주세요.
- 실제 노트 전사에 적합한 밀도로 작성해주세요.
```

## 빠르게 쓰는 짧은 버전

```md
[챕터명] 교안 PDF를 참고해서

- **개념별로 정리**하고
- 각 개념 옆에 **관련 페이지 범위**를 붙이고
- **손필기 표현, 수업식 강조 포인트, 시험 포인트**를 반영하고
- **임의로 중요한 내용을 빼지 말고**
- **이미지는 와이어프레임 형태로**
- **실제로 손으로 옮겨 적기 쉬운 md/txt 노트**를 생성해주세요.

특히 [중요 단원]은 절대 빼지 말아주세요.
문장은 짧게, 비교와 흐름 중심으로 정리해주세요.
```

## 사용 예시

```md
아래 자료들을 바탕으로 Chapter 3 Transport Layer 손필기 전사용 노트를 만들어주세요.

참고 자료:

- 교안 PDF: Chapter_3_v80_260324_091518.pdf

이번 작업의 핵심 목표는 다음과 같습니다.

- 개념별 구조 유지
- 각 개념 옆에 페이지 범위 표시
- 손필기 표현 반영
- 수업식 강조 포인트 반영
- RTT, timeout, ACK, fast retransmit, flow control, handshake, congestion control, AIMD, slow start, cwnd, rwnd는 절대 빼지 말 것
- 이미지는 와이어프레임 형태로
- 실제 손으로 노트 전사 가능한 밀도로 작성할 것

출력은 md + txt 파일 둘 다 부탁드립니다.
```
