# 개발 실력 향상을 위해…

막막함을 느끼는 지점은 매우 정상적인 단계입니다.

특히 **“AI 없이 구현해보자”**라는 단계에서 멈추는 것은 대부분 개발자 지망생이 겪는 공통 현상입니다. 문제의 원인은 보통 **능력이 부족해서가 아니라 구현 시작 전략이 없기 때문**입니다.

그래서 단순히 “예제를 따라치는 게 좋다/나쁘다”의 문제가 아니라 **어떤 방식으로 따라치느냐**가 핵심입니다.

---

# 1. 문제의 본질 구조

현재 막히는 이유는 다음 구조 때문입니다.

```
개념 이해
↓
코드 구현
```

많은 학습자가 이 두 단계만 있다고 생각합니다.

하지만 실제로는 **중간 단계가 하나 더 있습니다.**

```
개념 이해
↓
구조 설계
↓
코드 구현
```

지금 막막한 이유는 바로 **“구조 설계 단계”가 비어 있기 때문**입니다.

예를 들어 **Stack 구현**을 바로 코드로 시작하면 막힙니다.

하지만 구조를 먼저 만들면 훨씬 쉬워집니다.

---

# 2. 초보자가 자주 놓치는 설계 관점

자료구조나 간단한 기능을 구현할 때는 항상 먼저 **3가지 질문**을 해야 합니다.

```
1. 어떤 데이터가 필요한가?
2. 어떤 동작이 필요한가?
3. 어떤 예외 상황이 있는가?
```

예: Stack

```
데이터
- 리스트

동작
- push
- pop
- top
- size
- is_empty

예외
- pop할 때 비어 있음
```

이렇게 되면 **코드의 뼈대가 자동으로 만들어집니다.**

---

# 3. 예제 따라치기의 올바른 방법 (추천 루틴)

예제를 따라치는 것은 **초기 단계에서는 매우 좋은 학습 방법**입니다.

단, 반드시 아래 3단계를 포함해야 합니다.

### 1단계 — 이해하면서 따라치기

```python
class Stack:
    def __init__(self):
        self._data = []
```

이때 반드시 스스로 설명합니다.

```
왜 리스트를 썼지?
왜 _data 라는 이름일까?
```

---

### 2단계 — 파일 닫고 다시 구현

따라친 뒤 **IDE를 닫고 다시 작성합니다.**

예

```
Stack 클래스 만들기
push / pop 구현
```

이 단계에서 막히는 것이 정상입니다.

---

### 3단계 — 구조만 보고 구현

코드 대신 다음만 보고 구현합니다.

```
Stack

data
push
pop
top
is_empty
size
```

이 단계가 되면 **체화가 시작됩니다.**

# 4. 두 가지 학습 전략 비교

## 전략 A — 예제 중심 학습

```
예제 보기
→ 따라치기
→ 이해
```

장점

- 빠르게 익힘
- 안정적

단점

- 구현 능력 성장 느림

---

## 전략 B — 구현 중심 학습

```
문제 보기
→ 설계
→ 구현
→ 막힘
→ 참고
```

장점

- 문제 해결력 성장

단점

- 초반 스트레스 큼

# 5. 가장 추천하는 방식 (하이브리드)

가장 좋은 방식은 **A와 B의 혼합**입니다.

루틴 예시

```
1 예제 읽기
2 따라치기
3 IDE 닫기
4 다시 구현
5 막히면 참고
```

이 방식이 **구현 능력을 가장 빨리 올립니다.**

---

# 6. 실제 예시 (Stack)

예제를 따라친 뒤 이렇게 연습합니다.

### 구조

```
Stack
- data

push
pop
top
size
is_empty
```

### 구현

```python
class Stack:
    def __init__(self):
        self._data = []

    def push(self, value):
        self._data.append(value)

    def pop(self):
        return self._data.pop()

    def is_empty(self):
        return len(self._data) == 0
```

여기까지 **30분 안에 구현 가능**하면 체화 단계입니다.

# 7. 사용자의 현재 단계 평가

지금까지 대화를 보면 다음 단계에 있습니다.

```
개념 이해
→ 부분 구현
→ 체화 직전 단계
```

이미

- Stack 구현
- 예외 처리
- 테스트 코드
- Presenter 구조

까지 경험하셨습니다.

즉 **초보 초입 단계는 이미 벗어났습니다.**

지금 필요한 것은

```
반복 구현
```

입니다.
