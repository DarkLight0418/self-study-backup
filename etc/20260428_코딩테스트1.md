네, 지금 단계에서는 **문제를 잘 못 푸는 게 아니라 “문제를 푸는 고정 절차가 아직 몸에 안 붙은 상태”**에 가깝습니다.
그래서 더 어려운 문제를 많이 푸는 것보다, 아주 쉬운 문제로 **생각 순서 자체를 자동화**하는 게 먼저입니다.

핵심은 이겁니다.

> 쉬운 문제를 풀 때도 매번 같은 순서로 생각한다.
> 그러면 나중에 어려운 문제에서도 머리가 덜 멍해진다.

---

# 1) 지금 우선 목표는 “정답”이 아니라 “절차 고정”

지금은 이런 상태일 가능성이 큽니다.

```text
문제 읽음
→ 뭘 해야 할지는 어렴풋이 알겠음
→ 근데 코드로 옮길 순서가 안 잡힘
→ 머리가 멍해짐
→ 답을 보면 쉬워 보임
→ 다시 혼자 하면 막힘
```

이건 실력이 없어서라기보다, **문제 해석과 구현 사이를 연결하는 중간 단계**가 약해서 그렇습니다.

그 중간 단계가 바로:

```text
입출력 정리 → 예시 손풀이 → 규칙 찾기 → 의사코드 → 코드
```

입니다.

# 2) 가장 단순한 문제 풀이 절차

처음에는 이것만 고정하세요.

```text
1. 입력은 무엇인가?
2. 출력은 무엇인가?
3. 예시 하나를 손으로 바꿔본다.
4. 반복/조건/계산 중 무엇이 필요한지 고른다.
5. 한글 의사코드 3줄로 쓴다.
6. 코드로 옮긴다.
7. 예시로 다시 검산한다.
```

이 7단계를 매번 그대로 반복하면 됩니다.

처음에는 느려도 괜찮습니다.
중요한 건 **항상 같은 길로 들어가는 것**입니다.

# 3) 더 줄이면 “5문장 템플릿”으로 시작하세요

문제를 볼 때 매번 아래 5문장만 채우세요.

```text
1. 이 문제는 ________을/를 입력받는다.
2. 최종적으로 ________을/를 반환해야 한다.
3. 예를 들어 ________은/는 ________이 된다.
4. 그러려면 ________을/를 반복하거나 확인해야 한다.
5. 그래서 ________한 뒤 ________을 반환한다.
```

방금 문제인 `0 떼기`에 적용하면:

```text
1. 이 문제는 숫자로 이루어진 문자열을 입력받는다.
2. 최종적으로 앞쪽 0을 제거한 문자열을 반환해야 한다.
3. 예를 들어 "00123"은 "123"이 된다.
4. 그러려면 앞에서부터 문자가 '0'인지 확인해야 한다.
5. 그래서 0이 아닌 문자가 나오는 위치를 찾은 뒤 그 위치부터 잘라 반환한다.
```

이 정도면 이미 의사코드의 뼈대가 잡힌 겁니다.

---

# 4) 의사코드는 처음부터 멋있게 쓰려고 하지 않아도 됩니다

처음 의사코드는 코드처럼 정확할 필요 없습니다.
오히려 처음에는 **진짜 한글 행동 순서**로 쓰는 게 좋습니다.

예를 들어:

```text
앞에서부터 한 글자씩 본다.
0이면 계속 넘어간다.
0이 아니면 거기서부터 끝까지 잘라서 반환한다.
```

이게 1차 의사코드입니다.

그다음에 조금 더 코드에 가깝게 바꿉니다.

```text
i를 0으로 둔다.
n_str[i]가 '0'인 동안 i를 1씩 증가시킨다.
반복이 끝나면 n_str의 i번째부터 끝까지 반환한다.
```

그리고 마지막에 코드로 옮깁니다.

```java
int i = 0;

while (i < n_str.length() && n_str.charAt(i) == '0') {
    i++;
}

return n_str.substring(i);
```

즉, 의사코드는 한 번에 완성하는 게 아니라:

```text
느슨한 한글 → 코드에 가까운 한글 → 실제 코드
```

이 순서로 가면 됩니다.

# 5) 쉬운 문제에서 먼저 길러야 할 3가지

## 1순위: “문제 유형 이름 붙이기”

쉬운 문제를 볼 때마다 이렇게 분류하세요.

```text
이건 문자열 문제인가?
배열 문제인가?
반복문 문제인가?
조건문 문제인가?
누적합 문제인가?
개수 세기 문제인가?
최댓값/최솟값 찾기 문제인가?
```

`0 떼기`는 이렇게 볼 수 있습니다.

```text
문자열 문제 + 반복문 문제 + 조건 검사 문제
```

이름을 붙이면 머릿속이 덜 흐려집니다.

---

## 2순위: “대표 동작 하나 찾기”

문제 전체를 한 번에 보지 말고, 핵심 동작 하나만 찾습니다.

`0 떼기`의 대표 동작은:

```text
앞에서부터 0인지 확인한다.
```

이겁니다.

대표 동작이 잡히면 반복문이 보입니다.

```text
앞에서부터 확인한다 → 인덱스 i가 필요하다
0인 동안 계속 간다 → while문이 적절하다
멈춘 위치부터 자른다 → substring이 필요하다
```

---

## 3순위: “손으로 한 번 실행하기”

쉬운 문제일수록 손으로 안 풀고 바로 코드로 가려는 경우가 많습니다.
그런데 초반에는 손풀이가 더 중요합니다.

```text
"00123"

i = 0 → '0' → 증가
i = 1 → '0' → 증가
i = 2 → '1' → 멈춤
substring(2) → "123"
```

이렇게 적으면 코드가 거의 자동으로 나옵니다.

# 6) 추천 훈련법: 하루 3문제보다 “1문제 3단계”

지금은 문제 수보다 **풀이 절차 반복**이 중요합니다.

한 문제를 이렇게 3번 처리하세요.

## 1차: 말로 풀기

```text
이 문제는 무엇을 하라는 문제인가?
```

한 문장으로 씁니다.

예:

```text
문자열 앞에 붙은 0들을 제거하는 문제다.
```

---

## 2차: 손으로 풀기

예시를 하나 만듭니다.

```text
"00057" → "57"
```

그리고 어떻게 변하는지 적습니다.

```text
앞에서부터 0, 0, 0을 지나고
5부터 남긴다.
```

---

## 3차: 의사코드 쓰기

```text
i = 0
앞에서부터 0인 동안 i 증가
i부터 끝까지 반환
```

이렇게만 해도 충분합니다.

코드는 그다음입니다.

# 7) 아주 단순한 코딩 테스트 사고 체크리스트

문제 풀 때 옆에 이걸 켜두고 하세요.

```text
[문제 풀이 체크리스트]

1. 입력:
2. 출력:
3. 예시 변환:
4. 핵심 동작:
5. 필요한 도구:
   - 조건문?
   - 반복문?
   - 배열/문자열 인덱스?
   - 누적 변수?
6. 의사코드 3줄:
7. 코드:
8. 검산:
```

`0 떼기`로 채우면:

```text
1. 입력: 숫자로 이루어진 문자열
2. 출력: 앞의 0을 제거한 문자열
3. 예시 변환: "00123" → "123"
4. 핵심 동작: 앞에서부터 0인지 확인
5. 필요한 도구: 반복문, 문자열 인덱스, substring
6. 의사코드:
   - i를 0으로 둔다.
   - n_str[i]가 0인 동안 i를 증가시킨다.
   - i부터 끝까지 잘라 반환한다.
7. 코드:
8. 검산:
```

이렇게 하면 막연함이 많이 줄어듭니다.

너무 크게 잡지 말고, 오늘부터는 이렇게만 해보세요.

## 문제 하나당 반드시 10분은 코드 금지

처음 10분 동안은 코드 쓰지 않습니다.

대신 아래만 적습니다.

```text
입력:

```

이게 끝난 뒤에만 코드를 씁니다.

이 훈련이 중요한 이유는, 지금 필요한 게 “문법 반응속도”가 아니라 **문제를 코드로 바꾸는 중간 사고**이기 때문입니다.

# 9) 쉬운 문제를 고르는 기준

지금은 프로그래머스 기준으로 보면 다음 유형부터 추천합니다.

```text
문자열 자르기
문자열 반복
배열 원소 변환
특정 문자 세기
최댓값/최솟값 찾기
조건에 맞는 원소 고르기
합계 구하기
인덱스 찾기
```

이런 문제들은 쉬워 보여도 기본기를 만들기에 좋습니다.

초반에 피하면 좋은 유형은:

```text
DP
그래프
복잡한 재귀
수학 규칙이 강한 문제
정렬 + 조건이 꼬인 문제
```

아직은 이런 문제보다 **반복문/조건문/문자열/배열 감각**을 먼저 잡는 게 좋습니다.

# 10) 막힐 때 쓰는 문장

문제를 보다가 머리가 멍하면, 바로 이 문장을 쓰세요.

```text
내가 지금 해야 하는 일은 전체 정답을 떠올리는 게 아니라,
입력 하나가 출력으로 바뀌는 과정을 손으로 따라가는 것이다.
```

그리고 예시 하나만 잡습니다.

```text
"00123"이 왜 "123"이 되는가?
```

이 질문에 답하면 됩니다.

처음부터 코드가 아니라:

```text
앞의 0을 건너뛰고 1부터 남긴다.
```

이렇게 말로 답합니다.

이 말이 코드의 씨앗입니다.

# 11) 요약

지금은 “쉬운 문제도 못 떠올린다”가 아니라,
**문제를 코드로 바꾸는 절차가 아직 자동화되지 않은 상태**입니다.

그래서 당분간 목표는 이겁니다.

```text
정답 빨리 맞히기 X
풀이 절차 고정하기 O
```

가장 단순한 절차는:

```text
입력 확인
출력 확인
예시 손풀이
핵심 동작 찾기
의사코드 3줄
코드 작성
검산
```

그리고 의사코드는 처음부터 멋있게 쓰지 말고:

```text
느슨한 한글 → 코드에 가까운 한글 → 실제 코드
```

순서로 가면 됩니다.

오늘은 문제 하나를 많이 풀려고 하지 말고,
`0 떼기` 같은 쉬운 문제 하나를 가지고 **위 템플릿을 끝까지 채우는 것**만 해도 충분히 좋은 훈련입니다.
